  Application Security

Application security is the practice of ensuring the confidentiality, integrity, and availability of applications and their data. It involves implementing measures to prevent unauthorized access to applications and their data, as well as protecting against vulnerabilities that could be exploited by attackers.

Application security can include measures such as:

1️⃣ Input validation:
Ensuring that data entered into an application is in the correct format and does not contain any malicious code.

2️⃣ Access controls:
Implementing measures to ensure that only authorized users can access certain features or data within an application.

3️⃣ Encryption:
Protecting data within an application by encoding it so that it can only be accessed by authorized parties.

4️⃣ Security testing:
Regularly testing applications for vulnerabilities and implementing fixes to address any issues that are identified.

Effective application security is important because it helps to protect against cyber attacks and data breaches, and it helps to maintain the confidentiality, integrity, and availability of applications and their data.




  XSS

What is XSS?

Cross Site Scripting (XSS) is a vulnerability in a web application that allows injects malicious code into the vulnerable web application. The exploitation of XSS against a user can lead to various consequences such as account compromise, account deletion, privilege escalation, malware infection and many more. Cross site scripting attacks can be broken down into two types: stored and reflected.

✔️ Reflected XSS:

  SQL Injection

SQL injection is a type of cyber attack that involves injecting malicious code into an application's Structured Query Language (SQL) database. SQL is a programming language that is used to manage and manipulate data in relational database management systems (RDBMS).

In a SQL injection attack, an attacker injects malicious code into an application's SQL database by exploiting vulnerabilities in the application's input validation processes. This code is then executed by the database, allowing the attacker to gain unauthorized access to the database and potentially steal or manipulate data.

SQL statements by the attacker are designed so that they can be executed while the webserver is fetching content from the application server. It compromises the security of a web application.
Example:

So, this basically translates to:

STUDENT-ID == 12222345 or 1 = 1
Now this 1=1 will return all records for which this holds true. So basically, all the student data is compromised. Now the malicious user can also delete the student records in a similar fashion.

  Authentication Bypass

Authentication bypass is a type of cyber attack in which an attacker is able to gain access to a system or application without providing the correct credentials. This can be done by exploiting vulnerabilities in the authentication process or by using techniques such as brute force attacks or password cracking.

There are several ways that an attacker might attempt to bypass authentication, such as:

1️⃣ Guessing or cracking passwords
An attacker might try to guess or crack a user's password in order to gain access to a system or application.

2️⃣ Exploiting vulnerabilities in the authentication process:
An attacker might look for vulnerabilities in the way that an application or system handles authentication, such as weak passwords or unsecured login pages, and exploit these vulnerabilities to bypass authentication.

3️⃣ Using stolen or compromised credentials:
An attacker might use stolen or compromised user credentials to gain access to a system or application.

To prevent authentication bypass attacks, it is important to implement strong authentication measures, such as using strong passwords, enabling two-factor authentication, and regularly updating and patching systems and applications.





  OTP Bypass

One-Time Password (OTP) is a type of authentication method that involves the use of a temporary, single-use password to access a system or application. OTP is often used as an additional layer of security in addition to a username and password, to help prevent unauthorized access.

An OTP bypass attack is a type of cyber attack in which an attacker is able to gain access to a system or application by bypassing the OTP authentication process. This can be done by intercepting the OTP as it is being transmitted, using a stolen or compromised OTP, or by exploiting vulnerabilities in the OTP authentication process.

To prevent OTP bypass attacks, it is important to implement strong OTP authentication measures, such as using secure communication channels to transmit OTPs and regularly updating and patching systems and applications to fix any vulnerabilities. It is also important to educate users on the importance of keeping their OTPs secure and not sharing them with unauthorized parties.






  Rate Limit

Rate limiting is a process to limit requests possible. It is used to control network traffic. Suppose a web server allows up to 20 requests per minute. If you try to send more than 20 requests, an error will be triggered. This is necessary to prevent the attackers from sending excessive requests to the server. No rate limit is a flaw that doesn’t limit the no. of attempts one makes on a website server to extract data. It is a vulnerability which can prove to be critical when misused by attackers.

→ USES OF RATE LIMITING FEATURE:

1️⃣ Reduces excessive load on web servers

2️⃣ Prevent DOS (denial of service) attack

3️⃣ Help stop certain kinds of malicious bot activity like login to an account using multiple guess password and user id

4️⃣ Also prevent brute-force attacks

→How does rate limit work?

Rate limiting runs within an application, rather than running on the web server itself. Typically, rate limiting is based on tracking the IP addresses that requests are coming from and tracking how much time elapses between each request. The IP address is the main way an application identifies who or what is making the request.

A rate limiting solution measures the amount of time between each request from each IP address and measures the number of requests within a specified timeframe. If there are too many requests from a single IP within the given timeframe, the rate limiting solution will not fulfil the IP address's requests for a certain amount of time.

Essentially, a rate-limited application will say, "Hey, slow down," to unique users that are making requests at a rapid rate. This is comparable to a police officer who pulls over a driver for exceeding the road's speed limit, or to a parent who tells their child not to eat so much candy in such a short span of time.

  Open Redirect

An open redirect is a security flaw in an application or a web page that causes URLs to fail to authenticate properly. The open redirect is a failure in this phase that allows attackers to direct users to malicious websites of third parties. Open redirection happens when, via a user-controlled input, a web page is redirected to another URL in another domain. This happens when the program takes user-controlled data to the target of redirection in an unsafe way.

  CSRF

Cross-site request forgery (also known as CSRF) is a web security vulnerability that allows an attacker to induce users to perform actions that they do not intend to perform. It allows an attacker to partly circumvent the same origin policy, which is designed to prevent different websites from interfering with each other. For a CSRF attack to be possible, three key conditions must be in place:

→A relevant action. There is an action within the application that the attacker has a reason to induce. This might be a privileged action (such as modifying permissions for other users) or any action on user-specific data (such as changing the user's own password).

Cookie-based session handling. Performing the action involves issuing one or more HTTP requests, and the application relies solely on session cookies to identify the user who has made the requests. There is no other mechanism in place for tracking sessions or validating user requests.

 No unpredictable request parameters. The requests that perform the action do not contain any parameters whose values the attacker cannot determine or guess. For example, when causing a user to change their password, the function is not vulnerable if an attacker needs to know the value of the existing password.

  EXIF

EXIF stands for Exchangeable Image File Format and the EXIF data contains information such as the camera model and make, shutter speed, aperture, focal length, ISO number, date, time and much more. It can also store GPS coordinates of the location where an image was shot.

While being able to read such data is good to find out what settings were used to create a particular photograph, understanding photo metadata and how it can work against you is also important.

Even if you are not concerned about privacy, you would never want someone to get hold of sensitive personal information.

Many social media platforms remove the metadata from the images but there are still many images online where this information is available and can be easily extracted.





  HTML Injection

HTML injection is a type of cyber attack in which an attacker injects malicious HTML code into a website or web application. This can be done by exploiting vulnerabilities in the website or application, such as unvalidated input fields or unsecured forms.

Once the malicious HTML code has been injected, it can be used to modify the website or application in various ways, such as redirecting users to malicious websites, stealing sensitive information, or altering the website's content or appearance. HTML injection attacks can be used to launch other types of attacks, such as phishing attacks or malware infections.

To prevent HTML injection attacks, it is important to implement strong input validation measures and secure forms and input fields. Regular testing and patching of websites and applications can also help to identify and address any vulnerabilities that may be exploited by attackers. It is also important for users to be cautious when visiting websites and to be aware of any suspicious or unexpected changes to a website's content or appearance.





  Clickjacking

Clickjacking is an interface-based attack in which a user is tricked into clicking on actionable content on a hidden website by clicking on some other content in a decoy website.

For example, imagine an attacker who builds a web site that has a button on it that says, “click here for a free iPod”. However, on top of that web page, the attacker has loaded an iframe with your mail account and lined up exactly the “delete all messages” button directly on top of the “free iPod” button. The victim tries to click on the “free iPod” button but instead actually clicked on the invisible “delete all messages” button. In essence, the attacker has “hijacked” the user’s click, hence the name “Clickjacking”.





  SSRF

A Server-Side Request Forgery (SSRF) attack involves an attacker abusing server functionality to access or modify resources. The attacker targets an application that supports data imports from URLs or allows them to read data from URLs. URLs can be manipulated, either by replacing them with new ones or by tampering with URL path traversal.

One of the most prevalent examples of an SSRF attack is gaining access to Amazon EC2 instance credentials. If an IAM role is attributed to an EC2 instance, the provisional credentials may be accessed by completing a request to:

http://169.254.169.254/latest/meta-data/iam/security-credentials/{role-name} The level of damage that can be caused by the attacker depends on the level of access given to the IAM role. The higher the privileges of the role, the bigger the breach.





  LFI

A File Inclusion Vulnerability is a type of Vulnerability commonly found in PHP based websites and it is used to affect the web applications. This issue generally occurs when an application is trying to get some information from a particular server where the inputs for getting a particular file location are not treated as a trusted source.

It generally refers to an inclusion attack where an attacker can supply a valid input to get a response from a web server. In response, an attacker will be able to judge whether the input which he supplied is valid or not. If it is valid, then whatever/whichever file an attacker wants to see they can easily access it.

• Below example illustrates the working procedure of an LFI vulnerability:

     <?php   ‹// The Page we wish to display›

     int $file = $_GET['page']; ?>

Local File Inclusion Work: Let us understand the working of Local File Inclusion via the following example: In the above example "$file = $_GET['page'];" $file is just a file/code for indicating that the PHP code is going to return a ‘file’ in response. The $_GET is one of the sets of instruction which will get a particular file or webpage from a web server i.e. it will request for a file. The ['page'] is an argument that will define what type of pages will be displayed in response.
